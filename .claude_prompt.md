我需要修改 FitPlate Flutter 项目，将 NutritionReference 查表数据整合到个人资料页面作为营养推荐的依据。

## 背景
- `lib/data/nutrition_reference.dart` 中有 NutritionReference 类，包含基于身高、体重、性别的营养系数查表（针对不做力量训练人群）
- `lib/screens/profile_screen.dart` 是个人资料页面，已有身高、体重、性别、训练目标等字段
- 当用户在个人资料页勾选"不做力量训练（纯饮食控制）"时，应自动从 NutritionReference 表中查取推荐值

## 需要修改的文件

### 1. lib/data/nutrition_reference.dart
保持现有表格数据，添加便捷方法，在"不做力量训练"场景下给定性别/身高/体重，返回推荐的营养因子和说明文字。

### 2. lib/screens/profile_screen.dart
在已有的个人资料页面中整合 NutritionReference 查表功能。

## 具体实现

### 在 NutritionReference 中添加：

```dart
/// 获取推荐因子（只对不做力量训练的场景有意义）
/// 返回 (carbsPerKg, proteinPerKg) 或 null（无匹配）
static (double carbsPerKg, double proteinPerKg)? lookupRecommended({
  required bool isMale,
  required int heightCm,
  required int weightKg,
}) {
  final factor = getFactor(isMale: isMale, heightCm: heightCm, weightKg: weightKg);
  if (factor == null) return null;
  return (factor.$1, factor.$2); // getFactor 返回 (carbs, protein)
}

/// 获取推荐说明文字
static String recommendationNote({
  required bool isMale,
  required int heightCm,
  required int weightKg,
}) {
  final result = getFactor(isMale: isMale, heightCm: heightCm, weightKg: weightKg);
  if (result == null) return '当前身高/体重组合暂无参考数据';
  final daily = calculateDaily(factor: result, weightKg: weightKg.toDouble());
  final genderLabel = isMale ? '男' : '女';
  return '基于身高${heightCm}cm / 体重${weightKg}kg / $genderLabel 推荐：'
      '蛋白质 ${result.$2.toStringAsFixed(1)} g/kg（每日 ${daily.proteinG.toStringAsFixed(0)} g），'
      '碳水 ${result.$1.toStringAsFixed(1)} g/kg（每日 ${daily.carbsG.toStringAsFixed(0)} g）';
}
```

### 在 ProfileScreen 中修改：

1. 添加导入 `import '../data/nutrition_reference.dart';`

2. 在 _ProfileScreenState 中：
   - 添加 `String? _recommendationNote;` 状态变量，存储推荐说明文字
   - 添加 `(double carbsPerKg, double proteinPerKg)? _recommendedFactor;` 存储查表因子

3. 添加 `_lookupRecommendation()` 方法：
   ```dart
   void _lookupRecommendation() {
     final p = widget.profile;
     if (p.noStrengthTraining && p.goal == FitnessGoal.fatLoss) {
       final h = double.tryParse(_heightCtrl.text) ?? p.height;
       final w = double.tryParse(_weightCtrl.text) ?? p.weight;
       final result = NutritionReference.lookupRecommended(
         isMale: p.gender == Gender.male,
         heightCm: h.round(),
         weightKg: w.round(),
       );
       if (result != null) {
         _recommendedFactor = result;
         _recommendationNote = NutritionReference.recommendationNote(
           isMale: p.gender == Gender.male,
           heightCm: h.round(),
           weightKg: w.round(),
         );
         // 自动填入推荐值
         p.proteinPerKg = result.proteinPerKg;
         p.carbsPerKg = result.carbsPerKg;
         _proteinKgCtrl.text = p.proteinPerKg.toStringAsFixed(1);
         _carbsKgCtrl.text = p.carbsPerKg.toStringAsFixed(1);
       } else {
         _recommendedFactor = null;
         _recommendationNote = '当前身高/体重组合暂无参考数据';
       }
     } else {
       _recommendedFactor = null;
       _recommendationNote = null;
     }
   }
   ```

4. 在 initState 中调用 _lookupRecommendation() 来恢复已有数据

5. 修改"不做力量训练"开关的 onChanged 回调，开启时调用 _lookupRecommendation()，关闭时清除推荐数据

6. 在"每千克摄入量"卡片中，当 _recommendationNote 不为 null 时，在输入框下方添加一个 Container 显示推荐信息，带一个"应用推荐"按钮（如果用户已手动调整过，显示"恢复推荐"）

7. 当用户手动编辑蛋白质或碳水的输入框时，清除推荐状态（_recommendedFactor = null, _recommendationNote = null）

8. 在"每日营养目标"卡片中，如果使用了查表推荐，添加一个小提示图标/文字

注意：保持现有的 Material 3 风格，不要改动其他文件。修改后运行 `dart analyze` 验证没有编译错误。
